# 页面交互适配接口文档

更新时间：2026-04-28

## 1. 文档范围

本文档只整理当前三个页面联调需要的接口：

- 设计师主页：`src/views/DesignerOrders.vue`
- 全部订单：`src/views/AllOrders.vue`
- 商户管理：`src/views/MasterData.vue`

用户管理模块已联调完成，本文档不再覆盖用户管理接口。

当前页面代码仍以 mock 数据和前端内存操作为主，以下接口按“页面当前字段和交互”整理。

## 2. 通用约定

### 2.1 返回格式

```json
{
  "code": 200,
  "message": "操作成功",
  "data": {}
}
```

分页返回：

```json
{
  "code": 200,
  "message": "查询成功",
  "data": {
    "records": [],
    "total": 0,
    "pageNo": 1,
    "pageSize": 20
  }
}
```

### 2.2 时间格式

页面日期控件使用：

```text
YYYY-MM-DD HH:mm:ss
```

请求和响应都建议使用该格式，例如：

```text
2026-04-25 09:10:00
```

### 2.3 订单状态枚举

页面展示中文状态，接口建议同时返回状态编码和状态文案。

| status | statusName | 页面含义 |
| --- | --- | --- |
| UNASSIGNED | 未派单 | 调度已建单，未分配设计师 |
| UNCOMPLETED | 未完工 | 已分配设计师，设计师未点击完工 |
| PENDING_REVIEW | 待审核 | 设计师已提交完工，等待调度审核 |
| COMPLETED | 已完工 | 调度审核通过 |
| PROBLEM | 问题件 | 调度标记的问题订单 |
| OTHER | 其他 | 其他状态 |

### 2.4 查询条件传参方式

所有带筛选和分页的列表接口，前端直接向后端传一个对象，筛选条件全部放在 body 中。

建议列表查询统一使用 `POST`：

```http
POST /api/xxx/list
Content-Type: application/json
```

请求对象示例：

```json
{
  "pageNo": 1,
  "pageSize": 15,
  "shopIds": [1, 2],
  "productTypeNames": ["精修", "调色"],
  "statuses": ["UNASSIGNED", "UNCOMPLETED"],
  "designerIds": [8, 9],
  "startTime": "2026-04-01 00:00:00",
  "endTime": "2026-04-30 23:59:59",
  "keyword": "DD202604"
}
```

说明：

- 没选筛选条件时传空数组或不传该字段都可以，建议前后端统一为“空数组表示不过滤”。
- 时间范围未选择时，`startTime` 和 `endTime` 可以不传或传空字符串。
- 关键词为空时可以不传或传空字符串。

## 3. 设计师主页

页面路径：`/designer/orders`

页面文件：`src/views/DesignerOrders.vue`

### 3.1 页面交互

- 进入页面加载设计师本人订单列表。
- 支持商户多选、照片类型多选、订单状态多选、下单时间范围、关键词查询。
- 表格展示：商家名称、照片类型、状态、设计师、订单号、照片张数、客户信息、备注、下单时间。
- 点击表头“商家名称”打开商家明细弹窗，弹窗查询数据库所有订单数据，不分页，每个商家单独归类。
- 点击“完工”后，设计师提交该订单完工，订单进入待审核。
- 分页控件当前页面大小为 `17`。

### 3.2 查询设计师订单列表

设计师订单接口和调度的全部订单接口分开，不复用 `/api/orders`。

```http
POST /api/designer/orders/list
```

请求 body：

| 参数 | 类型 | 必填 | 页面来源 | 说明 |
| --- | --- | --- | --- | --- |
| pageNo | number | 否 | 分页 | 默认 1 |
| pageSize | number | 否 | 分页 | 当前页面默认 17 |
| shopIds | number[] | 否 | 商户多选 | 商户 ID 多选 |

| startTime | string | 否 | 时间范围 | 下单开始时间 |
| endTime | string | 否 | 时间范围 | 下单结束时间 |
| keyword | string | 否 | 关键词输入框 | 建议匹配订单号、客户信息、备注 |

返回字段：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | number/string | 后端生成的唯一数据 ID，后续完工操作使用 |
| orderNo | string | 订单号 |

### 3.3 设计师提交完工

只需要把当前行的唯一数据 ID 传给后端。

```http
POST /api/designer/orders/complete
```

请求示例：

```json
{
  "id": 1001
}
```

返回：更新后的订单对象或 `true`。

业务要求：

- 设计师只能提交分配给自己的订单。
- 只有 `UNCOMPLETED` 状态允许提交完工。
- 提交成功后状态变为 `PENDING_REVIEW`，页面展示“待审核”。

### 3.4 商家明细弹窗

弹窗查询数据库中所有订单数据，不分页，后端按商家归类后直接返回。前端不传筛选参数。

  ]

## 4. 全部订单

页面路径：`/orders`

### 4.1 页面交互

- 进入页面默认展示订单明细表。
- 支持商户多选、照片类型多选、订单状态多选、设计师多选、下单时间范围、关键词查询。
- 支持“详情 / 汇总”表格模式切换，切换时会清空关键词、清空勾选、页码回到第 1 页。
- 明细表支持跨页勾选，页面底部汇总已选订单的照片张数、接单合计、派单合计。
- 汇总表支持跨页勾选，页面底部汇总已选商户汇总行的订单数量、照片张数、接单总价、派单总价、利润。
- 导出按钮当前只有勾选数据后才启用，确定使用前端勾选导出。
- 明细表点击“商家名称”打开商家分组弹窗。
- 明细表点击“设计师”打开设计师分组弹窗。
- 明细表操作按钮按状态显示：派单、通过、退回、编辑、删除。
- 新建订单和编辑订单共用 `CreateOrder` 弹窗。

### 4.2 查询筛选下拉选项

合并接口，由后端分别从对应数据来源查询并拼成一个对象返回：

- 设计师数据来源：用户列表中所有角色为设计师的数据。
- 状态数据来源：订单状态固定枚举。

GET /api/orders/filter-options

### 4.5 新建订单

```http
POST /api/admin/orders
```

权限：调度、超级管理员。

当前 `CreateOrder` 弹窗字段：

| 页面字段 | 请求字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- | --- |
| 商家名称 | shopId | number/string | 是 | 页面建议选商户，提交 ID |
| 客户信息 | customerInfo | string | 是 | 当前页面是文本输入 |
| 照片类型 | productTypeName | string | 是 | 当前页面是文本输入 |
| 照片张数 | photoCount | number | 是 | 必须大于 0 |
| 接单价 | acceptUnitPrice | string/number | 是 | 不小于 0 |
| 派单价 | dispatchUnitPrice | string/number | 是 | 不小于 0 |
| 订单号 | orderNo | string | 是 | 唯一 |
| 下单时间 | orderedAt | string | 是 | `YYYY-MM-DD HH:mm:ss` |
| 备注 | remark | string | 否 | 备注 |

请求示例：

```json
{
  "orderNo": "DD20260425001",
  "shopId": 1,
  "customerInfo": "李小姐",
  "productTypeName": "精修",
  "photoCount": 8,
  "acceptUnitPrice": "12.00",
  "dispatchUnitPrice": "8.00",
  "orderedAt": "2026-04-10 09:10:00",
  "remark": "加急处理"
}
```

返回：订单详情对象，必须包含后端生成的唯一 `id`。

业务要求：

- 新建订单时不传设计师。
- 后端默认状态为 `UNASSIGNED`。
- 后端计算接单合计和派单合计。

  "shopId": 1,
  "customerInfo": "李小姐",
  "productTypeName": "精修",

  "dispatchUnitPrice": "8.00",
  "orderedAt": "2026-04-10 09:10:00",

业务要求：

- 当前页面点击“编辑”会复用新建订单弹窗并回填订单信息。

- 已完工订单允许编辑，但建议后端禁止修改商户、照片类型、设计师，保存后仍保持 `COMPLETED`。

```http
POST /api/admin/orders/dispatch
```

权限：调度、超级管理员。

### 4.9 审核通过和退回

审核通过和退回分成两个接口，都只传当前这条数据的唯一 ID 给后端。

审核通过：

```http
POST /api/admin/orders/approve
```

请求示例：

```json
{
  "id": 1001
}
```

退回：

```http
POST /api/admin/orders/reject
```

请求示例：

```json
{
  "id": 1001
}
```

业务要求：

- 审核通过后状态变为 `COMPLETED`。
- 退回后状态变为 `UNCOMPLETED`。
- 这两个接口主要处理 `PENDING_REVIEW` 状态数据。

### 4.11 导出已选数据

确定使用前端勾选导出。前端只把已勾选的行 ID 传给后端。

```http
POST /api/admin/orders/export
```

返回：`.xlsx` 文件流。

明细表导出请求：

```json
{
  "tableType": "DETAIL",
  "ids": [1001, 1002]
}
```

汇总表导出请求：

```json
{
  "tableType": "SUMMARY",
  "ids": ["summary-1-2026-04", "summary-2-2026-04"]
}
```

响应头：

```http
Content-Type: application/vnd.openxmlformats-officedocument.spreadsheetml.sheet
Content-Disposition: attachment; filename="orders.xlsx"
```

## 5. 商户管理

### 5.6 保存商户编辑

当前编辑弹窗最终仍需要一个保存接口，把商户 ID、修改后的商户名称、照片类型、客户信息一起提交给后端。

```http
PUT /api/merchant-master/merchants
```

请求示例：

```json
{
  "id": 1,
  "merchantName": "云帆摄影",
  "photoTypes": [
    {
      "photoType": "精修",
      "acceptPrice": "12.00",
      "dispatchPrice": "8.00"
    }
  ],
  "customers": [
    {
      "customerName": "李小姐",
      "photoCount": 18
    }
  ]
}
```

后端处理规则：

- 请求体中的 `id` 是商户 ID。
- 保存时以后端收到的 `photoTypes` 和 `customers` 数组为准，覆盖该商户原有照片类型和客户信息。
- 商户名称不能为空。
- 照片类型如果填写，则照片类型名称不能为空，默认接单价、默认派单价不能小于 0。
- 客户信息如果填写，则客户名称不能为空，照片张数必须大于 0。

### 6.2 当前页面需要补强的联调点

- 设计师订单接口和调度全部订单接口必须分开。
- 列表筛选统一使用 body 对象传参，方便前端直接提交当前筛选对象。
- 订单列表、汇总列表、商户列表返回都必须带后端生成的唯一 `id`。
- 商家/设计师分组弹窗使用后端专门接口，前端不传参，后端查询数据库所有数据并归类。
- 全部订单导出确定按前端勾选行导出，后端按传入 ID 生成文件。
- 商户新增时照片类型和客户信息可为空，后续通过编辑保存补充。
