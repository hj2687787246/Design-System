# 设计派单管理系统需求文档

## 1. 文档信息

- 文档版本：v0.1
- 创建日期：2026-04-24
- 文档状态：需求草案，待业务确认
- 项目目录：`C:\Users\Administrator\Desktop\design-dispatch-system`
- 适用范围：设计师、调度、超级管理员使用的设计建单、派单、完工与账号管理系统

## 2. 项目背景

系统用于管理店铺设计类订单的建单、派单、设计师完工提交、调度审核与账号权限管理。系统需要让调度可以高效创建店铺和订单，设计师只关注分配给自己的任务，超级管理员拥有完整业务权限和最高账号管理权限。

## 3. 角色与权限

系统共三类账号：

| 角色 | 数量限制 | 主要定位 | 权限摘要 |
| --- | --- | --- | --- |
| 设计师账号 | 多个 | 处理被派发的设计订单 | 查看本人相关订单、提交完工、修改本人密码 |
| 调度账号 | 多个 | 创建店铺、建单、派单、审核完工、回退状态 | 业务管理、店铺管理、设计师任务管理、修改本人密码 |
| 超级管理员 | 仅 1 个 | 老板账号，最高权限 | 拥有调度全部权限，额外可管理所有账号和重置所有账号密码 |

权限原则：

- 设计师只能看到与自己相关的数据。
- 调度可以看到所有店铺、订单、设计师派单和完工数据。
- 超级管理员可以看到并操作全系统数据。
- 超级管理员账号系统内只能存在一个，不允许创建第二个超级管理员。

## 4. 功能需求

### 4.1 登录鉴权

用户通过账号密码登录系统。登录成功后，系统根据角色返回可访问菜单、页面权限和数据权限。

需要支持：

- 账号密码登录。
- 登录失败提示。
- 登录状态保持。
- 退出登录。
- 未登录访问业务页面时跳转到登录页。
- 无权限访问页面时展示无权限提示，或跳转到当前角色可访问首页。

### 4.2 账号创建与账号管理

账号管理用于创建和维护系统用户。

设计师账号：

- 可以修改自己的登录密码。
- 不能创建账号。
- 不能查看其他账号信息。
- 不能修改其他账号密码。

调度账号：

- 可以修改自己的登录密码。
- 是否允许创建设计师账号待确认。
- 不能修改超级管理员账号。
- 是否允许修改设计师账号密码待确认。

超级管理员：

- 可以创建、查看、编辑、停用账号。
- 可以修改所有账号密码，包括设计师账号、调度账号和自己的账号。
- 可以重置普通账号密码。
- 不允许删除唯一超级管理员账号。

建议账号字段：

| 字段 | 说明 |
| --- | --- |
| 账号 ID | 系统唯一标识 |
| 登录账号 | 用户登录名，唯一 |
| 用户姓名 | 页面展示名称 |
| 角色 | 设计师、调度、超级管理员 |
| 手机号 | 可选 |
| 状态 | 启用、停用 |
| 创建时间 | 账号创建时间 |
| 更新时间 | 最近更新时间 |

### 4.3 店铺管理

调度和超级管理员可以创建和维护店铺。店铺用于建单时选择，并决定可选的商品类型和默认填充数据。

店铺创建需要支持：

- 创建店铺基本信息。
- 同时创建该店铺下的多个商品类型或单子类型。
- 为每个商品类型配置默认数据。
- 编辑店铺信息。
- 编辑商品类型和默认数据。
- 停用不再使用的店铺或商品类型。

建议店铺字段：

| 字段 | 说明 |
| --- | --- |
| 店铺 ID | 系统唯一标识 |
| 店铺名称 | 建单和列表展示使用 |
| 店铺负责人 | 可选 |
| 联系方式 | 可选 |
| 店铺状态 | 启用、停用 |
| 备注 | 可选 |

建议商品类型字段：

| 字段 | 说明 |
| --- | --- |
| 类型 ID | 系统唯一标识 |
| 所属店铺 | 关联店铺 |
| 类型名称 | 例如主图、详情页、海报等，具体枚举待确认 |
| 默认数据 | 建单选择该类型后自动填充到订单表单 |
| 类型状态 | 启用、停用 |

### 4.4 建单与默认数据填充

调度和超级管理员可以创建订单。

建单流程：

1. 进入建单页面。
2. 选择店铺。
3. 系统根据店铺加载该店铺下的商品类型。
4. 选择商品类型。
5. 系统读取该商品类型配置的默认数据，并自动填充到建单表单。
6. 调度补充或修改订单信息。
7. 上传相关附件或表单文件。
8. 保存订单。

建单表单建议字段：

| 字段 | 说明 |
| --- | --- |
| 订单编号 | 系统生成或手动填写，规则待确认 |
| 店铺 | 必填 |
| 商品类型 | 必填 |
| 商品名称 | 待确认是否必填 |
| 默认填充数据 | 来自商品类型配置，可编辑 |
| 设计要求 | 设计师执行说明 |
| 附件 | 表单、图片、文档等 |
| 截止时间 | 可选或必填，待确认 |
| 优先级 | 普通、加急等，待确认 |
| 订单状态 | 未完工、待审核、已完工 |
| 派单设计师 | 可为空，派单后必填 |
| 备注 | 可选 |

### 4.5 派单

调度和超级管理员可以将订单派发给设计师。

派单需要支持：

- 从未派单或需要改派的订单中选择设计师。
- 一个订单派给一个设计师。
- 是否支持一个订单派给多个设计师待确认。
- 派单后设计师可以在自己的任务表格中看到该订单。
- 改派时需要记录原设计师、目标设计师和操作人。

### 4.6 设计师任务处理

设计师登录后只能看到与自己相关的订单表格。

设计师表格支持：

- 关键词查询。
- 分页。
- 状态快速切换：全部、已完工、未完工。
- 查看订单详情。
- 对未完工订单点击完工按钮。

完工规则建议：

- 设计师点击完工后，订单进入待审核状态。
- 待审核状态下设计师不能回退为未完工。
- 调度或超级管理员审核通过后，订单变为已完工。
- 调度或超级管理员可以将已完工或待审核订单回退到未完工。

说明：如果业务希望设计师点击完工后直接变为已完工，则调度的“审核完工请求”功能会被弱化。建议保留待审核状态，便于调度确认。

### 4.7 完工审核与状态回退

调度和超级管理员可以审核设计师提交的完工请求。

审核流程：

1. 设计师提交完工。
2. 订单状态变为待审核。
3. 调度查看待审核列表。
4. 调度审核通过，状态变为已完工。
5. 调度审核不通过或回退，状态变为未完工，并可填写回退原因。

状态回退规则：

- 设计师不能回退自己已提交完工的数据。
- 调度可以将已完工数据回退到未完工。
- 超级管理员可以执行所有状态回退。
- 所有状态变更需要记录操作日志。

### 4.8 表格通用能力

系统主要业务数据以表格形式展示，所有角色可访问的表格都需要具备统一交互能力。

通用能力：

- 新增。
- 编辑。
- 删除或停用。
- 关键词查询。
- 分页。
- 状态快速切换：全部、已完工、未完工。
- 点击行查看详情。
- 操作按钮按角色权限展示。
- 加载中、空数据、查询失败状态提示。

设计师限制：

- 只展示本人相关数据。
- 只能对本人未完工任务执行完工操作。
- 不能新增、删除、派单、审核或回退。

调度和超级管理员：

- 可查看全量业务数据。
- 可建单、派单、审核、回退、编辑业务数据。

### 4.9 表格行详情

点击表格任意一条数据后，展示该条订单详情。

详情内容建议包括：

- 订单基本信息。
- 店铺信息。
- 商品类型信息。
- 默认填充数据。
- 派单设计师。
- 当前状态。
- 附件。
- 操作记录。
- 完工提交记录。
- 审核或回退记录。

设计师查看详情时，只展示本人任务相关信息，不展示其他设计师信息。

### 4.10 表头字段详情弹窗

表格表头字段支持点击查看该字段关联的汇总详情。

示例：点击“店铺名称”表头，弹窗展示所有店铺详情信息。

权限差异：

- 设计师点击表头详情弹窗时，只展示与自己相关的店铺和订单数据。
- 调度点击时，可以看到每家店铺下所有相关设计师和订单信息。
- 超级管理员与调度保持一致。

建议支持的表头详情：

| 表头字段 | 弹窗内容 |
| --- | --- |
| 店铺名称 | 店铺详情、店铺下商品类型、相关订单数量、相关设计师 |
| 商品类型 | 类型详情、默认数据、关联订单数量 |
| 设计师 | 设计师信息、派单数量、未完工数量、已完工数量 |
| 状态 | 各状态数量、状态说明、关联订单列表 |

## 6. 核心数据状态

建议订单状态设计：

| 状态 | 说明 | 可操作角色 |
| --- | --- | --- |
| 未完工 | 已创建或已派单，但设计师未提交完工 | 设计师可提交完工，调度/超级管理员可编辑或派单 |
| 待审核 | 设计师已提交完工，等待调度审核 | 调度/超级管理员可审核通过或回退 |
| 已完工 | 调度审核通过，订单完成 | 调度/超级管理员可回退 |

列表快速切换建议：

- 全部：展示全部状态。
- 已完工：只展示已完工。
- 未完工：展示未完工和待审核，或仅展示未完工，具体口径待确认。

## 7. 操作日志

为保证状态可追溯，以下操作需要记录日志：

- 登录成功、登录失败。
- 创建账号、编辑账号、停用账号、修改密码。
- 创建店铺、编辑店铺、停用店铺。
- 创建商品类型、编辑默认数据、停用商品类型。
- 建单、编辑订单、删除或停用订单。
- 派单、改派。
- 设计师提交完工。
- 审核通过。
- 审核不通过。
- 状态回退。

日志字段建议：

| 字段 | 说明 |
| --- | --- |
| 操作人 | 执行动作的账号 |
| 操作角色 | 设计师、调度、超级管理员 |
| 操作类型 | 建单、派单、审核等 |
| 操作对象 | 账号、店铺、订单等 |
| 操作前数据 | 可选 |
| 操作后数据 | 可选 |
| 操作时间 | 系统时间 |
| 备注 | 回退原因等 |

## 8. 非功能需求

权限安全：

- 前端根据角色控制菜单和按钮展示。
- 后端接口必须再次校验角色和数据范围。
- 密码不能明文存储。
- 超级管理员账号创建和数量限制必须由后端保证。

易用性：

- 设计师首页应直接展示“我的任务”。
- 调度首页应优先展示待派单、待审核、未完工数量。
- 表格查询条件需要保留，返回页面时不丢失当前筛选。
- 关键操作需要二次确认，例如删除、停用、回退、重置密码。

性能：

- 表格数据必须分页加载。
- 弹窗详情按需加载。
- 附件上传需要限制文件大小和类型，具体规则待确认。

兼容性：

- 后台管理端优先适配桌面浏览器。
- 是否需要移动端适配待确认。

## 9. 验收标准

- 设计师登录后只能看到本人相关任务。
- 设计师能对未完工任务提交完工，提交后不能自行回退。
- 调度能创建店铺、维护商品类型和默认数据。
- 调度建单时选择店铺和商品类型后，系统能自动填充默认数据。
- 调度能派单给设计师。
- 调度能审核设计师提交的完工请求。
- 调度能将已完工或待审核订单回退到未完工。
- 超级管理员拥有调度全部业务权限。
- 超级管理员能管理所有账号，并能修改所有账号密码。
- 所有角色表格均支持关键词查询、分页、状态快速切换和详情查看。
- 点击表头字段能打开对应详情弹窗，并根据角色限制数据范围。

## 10. 待确认问题

1. 调度账号是否可以创建设计师账号，还是只有超级管理员可以创建账号？
2. 调度账号是否可以修改设计师密码，还是只能修改自己的密码？
3. 设计师点击“完工”后，是直接变为已完工，还是进入待审核后由调度审核通过？
4. 列表“未完工”筛选是否包含待审核数据？
5. 一个订单是否只允许派给一个设计师，还是允许多个设计师协作？
6. 商品类型有哪些固定类型，还是由调度完全自定义？
7. 默认数据具体包含哪些字段，是否不同商品类型字段结构不同？
8. 建单上传附件支持哪些文件类型和大小限制？
9. 订单编号由系统自动生成，还是调度手动录入？
10. 是否需要订单截止时间、优先级、加急标记？
11. 是否需要删除能力，还是统一使用停用/作废以保留历史记录？
12. 系统是否只做 PC 后台，是否需要手机端适配？
