# 设计派单管理系统需求文档

## 1. 文档信息

- 文档版本：v1.0
- 更新日期：2026-04-25
- 文档状态：第一版完整需求整理稿
- 适用范围：调度、设计师、超级管理员使用的商户管理、建单、派单、完工审核、统计导出与账号权限管理系统

## 2. 项目背景

系统用于管理照片设计类订单的完整流转过程，包括商户资料维护、客户信息维护、照片类型价格配置、调度建单、调度派单、设计师加工、调度审核、订单统计与导出。

系统需要支持调度统一管理业务数据，设计师只处理分配给自己的订单，超级管理员负责账号创建和账号权限管理。

## 3. 角色与权限

系统包含三类账号：

| 角色 | 主要定位 | 权限摘要 |
| --- | --- | --- |
| 设计师 | 处理调度分配的设计订单 | 查看本人订单、筛选本人订单、查看详情、提交完工、修改本人账号密码 |
| 调度 | 管理商户、客户、订单、派单、审核和导出 | 新增商户、维护客户信息、建单、派单、审核完工、回退状态、编辑订单、导出数据、修改本人账号密码 |
| 超级管理员 | 账号与最高权限管理 | 创建设计师和调度账号、修改所有账号密码，并拥有和调度账号一样的业务操作权限 |

权限规则：

- 设计师和调度账号只能由超级管理员创建。
- 设计师和调度可以修改自己的账号密码，不能修改他人的账号密码。
- 超级管理员可以修改所有人的账号密码。
- 设计师只能看到分配给自己的订单数据。
- 调度可以看到全部商户、客户、订单、设计师派单、待审核和已完工数据。
- 超级管理员拥有和调度账号一样的业务操作权限。
- 调度和超级管理员可以导出表格文档。

## 4. 基础资料管理

### 4.1 商户管理

调度新增商户时需要填写：

| 字段 | 说明 |
| --- | --- |
| 商户名称 | 必填，建单和表格展示使用 |
| 照片类型 | 必填，可添加多个照片类型 |
| 默认接单价 | 每一种照片类型都需要配置默认接单价 |
| 默认派单价 | 每一种照片类型都需要配置默认派单价 |

规则：

- 每个商户可以配置多种照片类型。
- 每填写一种照片类型，都必须同时维护该类型的默认接单价和默认派单价。
- 商户用于调度建单时选择。
- 商户详情中需要展示该商户下的客户列表和相关订单信息。
- 商户不允许删除，也不允许停用。

### 4.2 客户信息管理

调度可以给指定商户添加客户信息。

添加客户信息时需要填写：

| 字段 | 说明 |
| --- | --- |
| 商户 | 必填，客户归属的商户 |
| 客户信息 | 必填，可理解为客户名称或客户详情 |
| 张数 | 必填，该客户对应照片张数 |

规则：

- 客户信息添加后归属到对应商户下面。
- 后续查看商户详情时，需要展示该商户下所有客户列表。
- 调度建单选择商户后，系统需要默认带出该商户下面的所有客户信息。
- 客户信息不允许删除，也不允许停用。

### 4.3 照片类型管理

照片类型需要支持通过接口动态添加。

规则：

- 照片类型不是固定写死枚举，需要支持后端接口返回。
- 调度可以在商户配置中给不同商户维护可用照片类型。
- 建单时照片类型从已维护的数据中选择。
- 选择照片类型后，系统应自动带出对应的默认接单价和默认派单价，并允许调度在建单时修改本单价格。
- 照片类型不允许删除，也不允许停用。

## 5. 调度建单与派单流程

### 5.1 建单字段

调度建单需要填写：

| 字段 | 说明 |
| --- | --- |
| 商家名称 | 必填，从已创建好的商户列表中勾选 |
| 客户信息 | 必填，选择商家后默认出现该商户下面所有客户信息 |
| 照片类型 | 必填，从照片类型列表中选择 |
| 张数 | 必填，参与金额合计计算 |
| 派单价 | 必填，可默认带出，也可手动调整 |
| 接单价 | 必填，可默认带出，也可手动调整 |
| 订单号 | 必填，调度手动输入，唯一性由后端校验 |
| 下单时间 | 必填 |
| 备注 | 可选 |

规则：

- 商家名称只能从已创建商户中选择。
- 选择商家名称后，客户信息区域默认展示该商户下面的所有客户信息，但每条订单只能选择一个客户。
- 选择照片类型后，默认带出该商户该照片类型配置的接单价和派单价。
- 接单价合计 = 张数 x 接单价。
- 派单价合计 = 张数 x 派单价。
- 订单号唯一性由后端校验，前端不做唯一性判断。
- 调度创建订单后可以不立即派单；如果尚未分配设计师，订单状态为未派单，并可一直停留在未派单状态。

### 5.2 派单规则

调度派单时，将订单分配给设计师。

规则：

- 调度选择设计师后，订单进入该设计师的表格数据中展示。
- 派单后订单状态变为未完工。
- 一个订单默认分配给一个设计师。
- 派单后不能修改设计师。
- 如果需要更换设计师，调度需要重新新增一条订单信息并选择新的设计师。
- 设计师只能看到分配给自己的订单。

## 6. 订单状态流转

订单状态共 6 种：

| 状态 | 说明 | 修改方式 |
| --- | --- | --- |
| 未派单 | 调度创建好了订单，但还没有分配给设计师 | 后端根据流程自动设置 |
| 未完工 | 调度已经派单给设计师，设计师完工前属于该状态，不包含未派单、待审核、已完工、问题件或其他状态 | 后端根据流程自动设置 |
| 待审核 | 设计师点击完工后进入该状态 | 后端根据流程自动设置 |
| 已完工 | 调度审核确认完工后进入该状态 | 后端根据流程自动设置 |
| 问题件 | 异常订单状态，不需要额外填写原因或备注 | 调度可以手动修改 |
| 其他 | 其他特殊状态，不需要额外填写原因或备注 | 调度可以手动修改 |

状态流转规则：

1. 调度建单后未派单：未派单。
2. 调度派单给设计师后：未完工。
3. 设计师点击完工后：待审核。
4. 调度确认完工后：已完工。
5. 调度回退状态后：重新变为未完工，并回到设计师和调度的未完工表格数据中。
6. 调度可以将订单手动标记为问题件或其他。

## 7. 首页表格与筛选

### 7.1 通用筛选条件

两种表格通用以下筛选规则：

| 筛选条件 | 说明 |
| --- | --- |
| 商家名称 | 支持按商家筛选 |
| 照片类型 | 支持按照片类型筛选 |
| 状态 | 支持按订单状态筛选 |
| 设计师 | 支持按设计师筛选 |
| 时间筛选 | 支持按订单时间筛选 |
| 关键词筛选 | 支持关键词模糊搜索 |

规则：

- 支持商家名称、照片类型、状态、设计师四个字段组合筛选。
- 时间筛选和关键词筛选需要可以与上述组合筛选同时使用。
- 表格支持分页。

### 7.2 第一种表格：订单明细表

第一种表格设计师和调度都可以使用，但展示字段按角色区分。

设计师表格展示字段：

| 字段 |
| --- |
| 商家名称 |
| 照片类型 |
| 状态 |
| 设计师 |
| 订单号 |
| 照片张数 |
| 客户信息 |
| 备注 |
| 下单时间 |

设计师表格分类筛选：

- 全部
- 已完工
- 未完工

调度表格展示字段：

| 字段 | 说明 |
| --- | --- |
| 商家名称 | 订单所属商户 |
| 照片类型 | 订单照片类型 |
| 状态 | 当前订单状态 |
| 设计师 | 已分配设计师 |
| 订单号 | 调度手动录入 |
| 照片张数 | 订单张数 |
| 接单价 | 单张接单价 |
| 接单价合计 | 张数 x 接单价 |
| 派单价 | 单张派单价 |
| 派单价合计 | 张数 x 派单价 |
| 客户信息 | 商户下客户信息 |
| 备注 | 订单备注 |
| 下单时间 | 订单下单时间 |
| 完工时间 | 调度确认完工时间 |

### 7.3 第二种表格：商户汇总统计表

第二种表格只有调度和超级管理员可以切换查看，设计师不能查看。

该表格需要按筛选条件对符合要求的数据进行汇总统计。
该表格按商户一行汇总，不按日期或其他维度拆分。

展示字段：

| 字段 | 说明 |
| --- | --- |
| 商家名称 | 汇总对象 |
| 订单数量 | 符合条件的订单数 |
| 照片张数 | 符合条件的照片总张数 |
| 接单总价 | 后端按符合条件的数据计算：张数 x 接单价 |
| 派单总价 | 后端按符合条件的数据计算：张数 x 派单价 |
| 利润 | 接单总价 - 派单总价 |
| 订单时间 | 格式为 YYYY-MM-DD HH:mm:ss |

规则：

- 统计结果必须以后端计算为准。
- 每个商户只展示一行汇总数据。
- 当前筛选条件会影响汇总结果。
- 订单时间展示格式固定为 YYYY-MM-DD HH:mm:ss。
- 调度或超级管理员导出第二种表格时，表头和数据以该汇总表为准。

## 9. 设计师完工与调度审核

### 9.1 设计师完工

设计师在表格的每条订单数据上可以点击完工。

规则：

- 设计师点击完工后，该订单状态变为待审核。
- 待审核订单进入调度的待审核列表。
- 设计师点击完工后，不能自行确认已完工，也不能自行回退状态。

### 9.2 调度待审核列表

调度待审核列表需要支持以下操作：

| 操作 | 说明 |
| --- | --- |
| 回退状态 | 将订单改回未完工流程，继续出现在设计师和调度两种账号的未完工表格数据中 |
| 确认完工 | 将订单添加到设计师和调度两种账号的已完工列表中 |
| 编辑 | 除商户名称、照片类型、设计师外，其他订单数据都可以修改并重新保存 |

规则：

- 回退状态后，订单重新进入设计师和调度的未完工数据中，状态按流程回到未完工。
- 确认完工后，订单状态变为已完工，并记录完工时间。
- 编辑时不能修改商户名称、照片类型、设计师三个字段。
- 编辑时除商户名称、照片类型、设计师外，其他订单业务数据都可以修改。
- 编辑保存后，该订单依旧保持已完工状态。
- 编辑保存后，设计师和调度表格中都需要展示最新修改后的信息。

## 10. 导出功能

调度和超级管理员可以导出文档。

导出规则：

- 当前选择第一种表格时，导出的表头和数据按订单明细表定义。
- 当前选择第二种表格时，导出的表头和数据按商户汇总统计表定义。
- 导出数据为当前表格中已勾选的数据。
- 导出格式只能为 Excel 文档。
- 设计师没有导出入口。

## 12. 非功能需求

权限安全：

- 前端需要根据角色控制菜单、按钮和字段展示。
- 后端接口必须校验角色权限和数据范围，不能只依赖前端隐藏。
- 设计师不能通过接口访问他人订单。
- 设计师不能查看仅调度可见的价格字段，例如设计单价。

易用性：

- 调度建单时，选择商户后自动加载客户信息和照片类型配置。
- 表格筛选条件需要保留，返回页面后不应丢失当前筛选状态。
- 回退状态、确认完工、修改价格、导出等关键操作需要二次确认。
- 商户、客户和照片类型不提供删除或停用操作入口。

性能：

- 表格必须分页加载。
- 汇总统计应由后端完成计算。
- 详情数据按需加载。

## 13. 验收标准

- 调度可以新增商户，并为每种照片类型配置默认接单价和默认派单价。
- 调度可以给商户添加客户信息和张数，且客户归属到该商户下面。
- 照片类型可以通过接口动态添加和获取。
- 调度建单时可以选择已创建商户，并自动展示该商户下所有客户信息，但每条订单只能选择一个客户。
- 调度建单时可以填写照片类型、张数、派单价、接单价、订单号、下单时间和备注。
- 订单号唯一性由后端校验，前端不需要提前判断唯一性。
- 调度可以创建订单后暂不派单，订单一直停留在未派单状态。
- 调度派单后，订单能直接出现在对应设计师的表格数据中。
- 派单后不能修改设计师；如需更换设计师，需要重新新增订单并选择新的设计师。
- 两种表格都支持商家名称、照片类型、状态、设计师、时间和关键词组合筛选。
- 订单明细表支持分页，并按角色展示不同字段。
- 商户汇总统计表只有调度和超级管理员可见，按商户一行汇总，并能正确计算订单数量、照片张数、接单总价、派单总价和利润。
- 商户名称字段详情按角色展示不同只读信息。
- 设计师字段详情中，设计单价只对调度展示。
- 设计师点击完工后，订单进入调度待审核列表。
- 调度可以在待审核列表中回退状态、确认完工和编辑订单。
- 回退状态后，订单重新进入未完工流程。
- 确认完工后，订单进入已完工列表并记录完工时间。
- 编辑已完工订单时不能修改商户名称、照片类型、设计师，保存后订单保持已完工状态，且设计师和调度表格展示最新数据。
- 订单状态完整支持未派单、未完工、待审核、已完工、问题件、其他。
- 只有超级管理员可以创建设计师和调度账号。
- 设计师和调度只能修改自己的账号密码，超级管理员可以修改所有人的账号密码。
- 超级管理员拥有和调度账号一样的业务操作权限。
- 调度和超级管理员可以导出 Excel 文档，导出表头根据当前选择的表格类型确定，导出数据为表格勾选中的数据。

## 14. 已确认规则

1. 问题件和其他状态不需要填写原因或备注。
2. 商户、客户、照片类型不允许删除，也不允许停用。
3. 订单号唯一性由后端校验，前端不做唯一性判断。
4. 订单时间展示格式为 YYYY-MM-DD HH:mm:ss。
5. 导出格式只能为 Excel 文档。
6. 每条订单只能选择一个客户。
7. 产品字段不需要，后续开发不实现该字段。
8. 调度创建订单后允许不派单，未派单订单会一直停留在未派单状态。
9. 派单后不能修改设计师；如果需要更换设计师，需要调度重新新增订单并选择新的设计师。
10. 需求中的未完工只表示设计师已接单但未点击完工的状态，不包含其他状态。
11. 调度编辑订单时，除商户名称、照片类型、设计师三个字段不能修改外，其他字段都可以修改。
12. 商户汇总统计表按商户一行汇总。
13. 导出的 Excel 只包含当前表格勾选中的数据。
14. 超级管理员拥有和调度账号一样的业务操作权限，包括导出权限。
